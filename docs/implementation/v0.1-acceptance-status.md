# v0.1 acceptance status

## Purpose

This document maps every [v0.1 acceptance criterion](../product/scope.md#acceptance-criteria) to executable evidence and remaining human review. It is a release-gate ledger, not a release declaration.

The technical scenario suite currently contains ten content-addressed cases: seven feasible plans and three explicit infeasibility outcomes. All ten conform to their pinned expectations. None has completed independent domain-expert review. Therefore v0.1 is **not release eligible**.

## Status vocabulary

- **Technical pass:** executable repository evidence currently satisfies the technical part of the criterion.
- **Review required:** repository evidence exists, but a qualified human must judge its safety, usefulness or completeness.
- **External gate:** the criterion cannot be completed by the implementation author or CI.

## Reproduce the technical evidence

From the repository root:

```bash
make check

go run ./cmd/yara scenario validate-all scenarios/v0.1 \
  --audit-output v0.1-scenario-suite.audit.jsonl

go run ./cmd/yara audit verify v0.1-scenario-suite.audit.jsonl
```

The suite output must report:

```text
requiredScenarioCount: 10
scenarioCount: 10
technicallyConformant: 10
planned: 7
infeasible: 3
technicalCoverageComplete: true
independentReviewsComplete: 0
independentReviewStatus: required
releaseEligible: false
```

`releaseEligible` remains false even after technical success because review evidence is intentionally outside the CLI's trust boundary.

## Human review evidence

Follow [the independent-review procedure](../../scenarios/REVIEWING.md). Each scenario directory contains a `review.md` packet with the exact identities that must be reviewed. A verdict is valid only for those identities; changing a request, inventory, catalog, expectation or resulting plan invalidates it.

The current Markdown review packets are deliberately not machine-counted. A future signed `ScenarioReview` resource needs a separate schema, identity-assurance model and ADR before automated release-gate counting is safe.
