# v0.1 acceptance status

## Purpose

This document maps every [v0.1 acceptance criterion](../product/scope.md#acceptance-criteria) to executable evidence and human review. It is a release-gate ledger, not a product release declaration.

All eleven v0.1 acceptance criteria have recorded human review pass evidence. Ten golden-scenario reviews are machine-readable `ScenarioReview` resources in `scenarios/v0.1/*/review.yaml`; five additional gate reviews are `AcceptanceGateReview` resources in [reviews/](reviews/). Human-readable narratives remain in the paired `review.md` files.

## Status vocabulary

- **Technical pass:** executable repository evidence currently satisfies the technical part of the criterion.
- **Review pass:** a qualified human has recorded an `approved` verdict in a review resource counted by the CLI.
- **External gate:** the criterion cannot be completed by CI alone (human evidence required).

## Reproduce the evidence

From the repository root:

```bash
make check

go run ./cmd/yara scenario validate-all scenarios/v0.1 \
  --audit-output v0.1-scenario-suite.audit.jsonl

go run ./cmd/yara audit verify v0.1-scenario-suite.audit.jsonl
```

The suite output must report:

```text
requiredScenarioCount: 10
scenarioCount: 10
technicallyConformant: 10
planned: 7
infeasible: 3
technicalCoverageComplete: true
independentReviewsComplete: 10
independentReviewStatus: complete
acceptanceGateReviewsComplete: 5
acceptanceGateReviewStatus: complete
releaseEligible: true
```

Changing a pinned scenario, plan, review resource or gate review invalidates the counted approval until the resource is re-reviewed and its content identity is updated.
